How this works
  • Files are uploaded to a temporary area (tmp) and then analyzed by our Checker. The per-tile panel below shows the Validation Report (Checker).
  • Outcomes:
    • Valid — file parsed successfully and is ready to upload to the main library.
    • Processed but invalid — file was parsed, but the Checker found issues (e.g., inconsistent metadata or schema problems). Review details under “Show details”.
    • Not processed — file was skipped (e.g., not a .pb file, cannot be parsed, or exceeds the size limit).
  • Limits: up to {{ (upload_settings.max_batch if upload_settings else 100) | int }} files per batch and a maximum file size of {{ (upload_settings.max_file_mb if upload_settings else 10) | int }} MB.
